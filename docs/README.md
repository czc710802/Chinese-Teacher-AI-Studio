# 项目文档索引

`docs/` 是本项目的唯一可信来源（Source of Truth）。若对话记录与本文档冲突，以本文档为准。

## 文档维护规则

1. 新需求先更新相关文档，再实施代码变更。
2. 每个重要任务完成后，同步更新相关设计文档与 `CHANGELOG.md`。
3. `CHANGELOG.md` 必须记录日期时间、修改内容、修改原因、涉及文件和版本号。
4. 项目恢复时，优先阅读本目录中的 Markdown 文档，不依赖聊天记录。
