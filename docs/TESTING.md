## 2026-06-22 手机端登录测试

- **测试环境：** 390×844 手机视口，访问 `http://localhost:5173/login`。
- **界面结果：** 登录页布局、账号/密码输入框和登录按钮均可正常显示。
- **教师账号结果：** 使用 `teacher / 123456` 提交后页面提示 `Failed to fetch`，未能进入教师端。
- **结论：** 当前手机端登录不通过，阻塞点在 API 服务不可达，不是移动端页面布局问题。
- **已定位原因：** `.env.local` 将服务端 `PORT` 配置为 `5173`，与前端开发端口冲突；同时客户端在 `5173` 开发模式固定请求 `http://<host>:4000/api`，导致登录请求无法到达后端。
- **修复：** 服务端端口已恢复为 `4000`，前端保持 `5173`。
- **复测结果：** 修复后教师账号可进入 `/teacher`，学生账号可进入 `/student`；两项均在 390×844 手机视口通过。
