# 思维教练增强设计

## 目标

将高中作文 AI 批改 App 从“给分和指出问题”升级为“高中作文思维训练系统”。AI 需要围绕诊断、提问、引导、修改、再评价形成闭环，让学生知道为什么错、如何追问、怎样修改到一类文水平。

## 范围

本轮增强现有作文批改链路，不新建独立产品。批改结果 JSON、学生端结果页、学生档案和教师端班级分析同步扩展，旧批改数据通过前端兜底保持可读。

## 数据结构

AI 批改结果在 `raw_json` 中新增：

- `logic_thinking_score`：30 分逻辑思维能力分析，包含观点明确、论证结构、推理能力、材料使用、论证深度 5 个子项。
- `thinking_depth`：星级、标签、所处层级、原因说明。
- `thinking_improvement`：从“是什么”推进到“为什么、本质、社会、时代、人性”的追问建议。
- `socratic_questions`：苏格拉底式追问列表，避免直接给答案。
- `thinking_coach`：诊断、提问、引导、修改任务、再评价标准的闭环步骤。

## 学生端

`ReviewPage` 新增“思维教练”模块，展示：

- 逻辑思维能力 30 分总览和五项子分。
- 思维深度星级和当前层级。
- 思维提升建议。
- 苏格拉底式追问。
- 深度修改闭环。

学生档案 `StudentProfile` 新增思维成长统计，展示逻辑能力、思辨能力、论证能力、材料分析能力、语言表达能力、修改能力。

## 教师端

班级分析接口从批改 `raw_json` 中提取逻辑思维子项和思维层级，生成：

- 全班思维薄弱项。
- 班级思维能力平均水平。
- 针对性教学建议。

教师端显示“班级思维分析”，用于指导课堂讲评。

## 降级

历史批改或模型异常导致字段缺失时，前端使用兜底结构。离线演示批改 `mockReview` 必须同步包含新字段，防止系统退回浅层反馈。

## 验证

- `server/test/ui-flow.test.js` 覆盖提示词、前端模块、教师端入口。
- `server/test/student-access.test.js` 或专用测试覆盖学生成长档案。
- `server/test/production-data.test.js` 或 analytics 测试覆盖班级思维分析。
- `pnpm --dir client build` 或项目根 `npm run build` 验证前端构建。
