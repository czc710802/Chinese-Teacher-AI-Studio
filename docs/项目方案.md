## 2. 数据库表结构

已在 `server/src/db/schema.js` 实现以下 SQLite 表：

| 表名 | 用途 |
|---|---|
| users | 登录账号、角色、姓名 |
| students | 学生扩展信息 |
| teachers | 教师扩展信息 |
| classes | 班级与学生关系管理 |
| class_students | 班级学生关系 |
| assignments | 作文任务、类型、满分、截止时间 |
| essays | 作文原文、修改稿、提交轮次 |
| essay_images | 作文图片与 OCR 文本 |
| ai_reviews | AI 批改结果 JSON 与拆分字段 |
| teacher_comments | 教师二次批注 |
| student_profiles | 学生作文成长档案 |
| export_records | Word/PDF 导出记录 |

## 3. 后端 API 路由

| 方法 | 路径 | 功能 |
|---|---|---|
| POST | `/api/auth/login` | 学生、教师登录 |
| GET/POST | `/api/classes` | 查看/创建班级 |
| GET | `/api/classes/:id/students` | 查看班级学生 |
| DELETE | `/api/classes/:classId/students/:studentId` | 删除班级学生 |
| GET/POST | `/api/assignments` | 查看/发布作文任务 |
| GET/POST | `/api/essays` | 查看/提交作文并触发 AI 批改 |
| POST | `/api/essays/ocr` | 多图 OCR 识别并合并文字 |
| GET | `/api/essays/:id` | 作文详情与批改结果 |
| POST | `/api/essays/:id/comments` | 教师二次批注 |
| GET | `/api/analytics/classes/:classId` | 班级作文数据分析 |
| GET | `/api/analytics/students/:studentId` | 学生成长档案 |
| POST | `/api/reports/essay/:essayId/:format` | 导出单篇作文报告 |
| POST | `/api/reports/student/:studentId/:format` | 导出学生成长档案 |
| POST | `/api/reports/class/:classId/:format` | 导出班级分析报告 |

## 5. AI 批改 Prompt

Prompt 写在 `server/src/services/prompt.js`。核心要求：

- 以高中语文特级教师、高考作文阅卷专家身份批改。
- 依据课程标准、高考作文评分思路。
- 覆盖审题立意、内容充实度、结构层次、论证逻辑、材料运用、语言表达、文采亮点、错别字、病句、卷面与规范表达、综合评分。
- 输出 JSON：总分、分项得分、等级、优点、问题、逐段点评、可修改句子、建议、升格段落、好句、下一步训练建议。

## 6. 图片识别 OCR 方案

后端 `/api/essays/ocr` 支持多张作文图片上传。服务端把图片转为 base64 data URL，调用 OpenAI Responses API 的视觉输入能力，按图片顺序合并为作文正文。没有 `OPENAI_API_KEY` 时返回示例文本，保证最小版本可演示。

## 7. Word/PDF 导出方案

- Word：使用 `docx` 生成 `.docx`，适合后续教师编辑。
- PDF：使用 `pdfkit` 生成 `.pdf`，优先加载 macOS 中文字体，保证中文可读。
- 导出范围：单篇作文批改报告、学生成长档案、班级作文分析报告。
- 导出记录写入 `export_records`。
