## 2026-06-21 22:50：修复 AI 辅导老师无法回复
### 问题
AI 辅导老师（/api/ai/tutor/chat）在浏览器中发送问题后无回复。

### 原因
1. `callTextModel()` 函数在调用 DeepSeek 时全局固定了 `response_format: { type: 'json_object' }`。
2. DeepSeek 的 `json_object` 模式要求 prompt 中必须包含 "json" 关键字，否则 API 返回错误。
3. `tutorChat()` 函数返回的是自然语言文本，prompt 中没有 "json" 关键字，导致 DeepSeek API 报错。

### 解决
1. 修改 `server/src/services/openai.js`：
   - `callTextModel(prompt)` → `callTextModel(prompt, options = {})`，新增可选 `options.jsonMode` 参数
   - `callDeepSeek(prompt)` → `callDeepSeek(prompt, options = {})`，只在 `options.jsonMode` 为 true 时才设置 `response_format: { type: 'json_object' }`
   - `reviewEssay()` 中的调用改为 `callTextModel(prompt, { jsonMode: true })`
2. 修改 `server/src/services/ai-tutor.js`：
   - `generateWritingExercise`、`upgradeEssay`、`mockMark`、`arbitrateMark`、`generateDailyBriefing`、`generateClassInsight` 的 `callTextModel(prompt)` 全部改为 `callTextModel(prompt, { jsonMode: true })`
   - `tutorChat` 保持 `return callTextModel(prompt)` 不变（无 jsonMode）

### 验证
- curl 测试 AI 辅导老师：AI 正常回复 400+ 字的建议 ✅
- 所有 JSON 输出的 function 的 prompt 中都包含 "JSON" 或 "json" 关键字 ✅
- 公网 https://pi.zhenwanyue.icu 可正常访问 ✅

### 注意事项
- DeepSeek 的 `json_object` response_format 模式有**两个限制**：
  (1) prompt 必须包含 "json" 关键字
  (2) 只能用于输出 JSON 的场景，不能用于自然语言对话
- 因此 `callTextModel` 不能全局固定 jsonMode，需要按需传入
- 后续新增 AI 功能时要注意选择正确的调用模式
